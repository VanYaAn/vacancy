# Проект: Поиск вакансий и генерация резюме

## Цель
Автоматизировать поиск вакансий через API hh.ru, хранение данных и генерацию персонализированных резюме.

## Архитектура

- **Go-приложение** — главный оркестратор всего пайплайна
- **PostgreSQL** — хранение вакансий и сопутствующих данных
- **Google Sheets** — финальное хранилище для анализа и удобной работы с данными
- **Python (через MCP-сервер)** — генерация резюме по шаблону
- **LLM (через MCP-сервер)** — адаптирует резюме под конкретную вакансию на основе её контента
- **MCP-серверы** — связующий слой между компонентами

## Пайплайн

1. Поиск вакансий через API hh.ru по заданным критериям
2. Сохранение вакансий в PostgreSQL
3. Экспорт данных из PostgreSQL в Google Sheets
4. Передача данных вакансии в MCP-сервер
5. LLM анализирует контент вакансии и адаптирует резюме под неё по шаблону
6. Сборка итогового резюме из релевантных частей шаблона

## Технологии

| Компонент       | Технология        |
|-----------------|-------------------|
| Оркестрация     | Go                |
| База данных     | PostgreSQL        |
| Анализ/экспорт  | Google Sheets API |
| Генерация резюме | Python (MCP)     |
| Интеграции      | MCP-серверы       |
| Источник данных | hh.ru API         |

## Генерация резюме (детально)

1. Go-оркестратор вызывает MCP-сервер, передавая данные конкретной вакансии
2. MCP-сервер передаёт контент вакансии + шаблон резюме в LLM
3. LLM анализирует требования вакансии и выбирает/адаптирует релевантные блоки из шаблона
4. Итоговый документ сохраняется и привязывается к записи вакансии

## Заметки

- Критерии поиска вакансий будут описаны отдельно
- Шаблон резюме (блоки, из которых LLM собирает итоговый вариант) будет описан отдельно
